# LifeOS — 身份优先的个人转型与生命管理

> 目的：让每一次开发都对齐同一个终点。本文件定义 LifeOS 的最终形态、用户旅程、核心模块、交互机制与信息层级，作为产品与设计的北极星。

## 1. 最终目标（North Star）

LifeOS 不是待办清单，也不是普通日记，而是一个**“身份优先 + 控制论反馈 + 游戏化驱动”的人生操作系统**。它帮助用户：
- 明确“我是谁/我正在成为谁”，并把身份变成每天可执行的行动结构。
- 用**控制论反馈回路**持续纠偏：行动 → 感知 → 比较 → 调整。
- 把生活游戏化为**主线任务、Boss 战、日常任务与经验值**，以进度感和回报持续推进转型。

## 2. 核心世界观与原则

1) **身份优先（Identity First）**
- 不改变身份认同，任何改变都会回弹。
- “我是那样的人”是系统中最高优先级。

2) **反愿景（Anti‑Vision）**
- 不仅要知道想要什么，还要明确“如果不改变会变成什么”。
- 反愿景是系统的风险提示与压力引擎。

3) **行为目的论（Teleology）**
- 每一个拖延/回避行为都有目的（安全感/避免评价）。
- 纠偏必须先解释动机，再调整行动。

4) **控制论反馈（Cybernetics）**
- 系统要像恒温器一样纠偏：持续对比“当前状态 vs 目标身份”。

5) **人生游戏化（Life as a Game）**
- 愿景 = 胜利条件；反愿景 = 失败代价。
- 1年目标 = 主线任务；1月项目 = Boss 战；每日行动 = XP 任务。

## 3. 产品最终形态（Life Console）

**Life Console = 身份数据库 + 游戏化任务系统 + 控制论纠偏引擎 + 中断机制 + 复盘引擎**

### 3.1 身份数据库（Identity Database）
- **反愿景库**：具象、感官化的“5‑10 年后如果不改变的生活”
- **愿景 MVP**：最低可行理想身份的声明（“我是那种……的人”）
- **约束条件**：绝不牺牲的底线（规则）

### 3.2 游戏化目标架构（Gamified Teleology）
- **主线任务（1年目标）**：唯一的胜利条件
- **Boss 战（1月项目）**：推进主线的技能/项目
- **每日任务（Daily Quests）**：直推 Boss 的每日杠杆行动
- **积分/经验值（XP）**：行为即积分

### 3.3 控制论纠偏引擎（Cybernetic Correction）
- 未完成任务时，系统不提示“延期”，而是触发**目的论追问**：
  - “你没做，是为了保护什么？”
  - “你在逃避什么评价？”
- AI 必须给出**下一步可验证行动**，而非泛化建议。

### 3.4 中断机制（Interrupter）
- **随机/定时反思打断**（11:00 / 15:00 / 19:00 或随机）
- 提问不是“提醒做事”，而是“打断自动驾驶”。

### 3.5 洞察与复盘引擎
- 晚间总结“存活时刻 / 麻木时刻”
- 指向真正的内部敌人（信念/回避）

## 4. 用户旅程（1‑Day Reset Protocol）

### 阶段 1：早晨 — 心理挖掘
- 引导式写作生成：反愿景 + 身份声明 + 愿景 MVP

### 阶段 2：白天 — 打断自动驾驶
- 系统中断提问，迫使用户觉察当下行为目的

### 阶段 3：夜晚 — 结构化整合
- 把愿景 → 1年目标 → 1月Boss → 每日任务转为层级结构
- 确认主线与明日行动

## 5. 日常使用循环（Daily Loop）

- **晨间启动**：重读身份声明，回答“如果我是那个人，我今天会做什么？”
- **日间纠偏**：中断提问，意识觉醒
- **晚间结算**：完成日常任务、计算 XP、更新链条

## 6. 信息架构与界面结构

### 6.1 视觉隐喻：同心圆力场
- 中心：愿景 / 身份
- 第二层：1年主线
- 第三层：1月 Boss
- 最外层：今日任务

### 6.2 透镜视角（Lens）
- 年度透镜 / 月度透镜 / 今日透镜
- 任何层级都必须能看到反愿景作为“失败代价”背景

### 6.3 风格基调
- **克制、冷静、严肃的游戏感**
- 允许深色背景与轻微压迫感，但避免娱乐化、可爱化

## 7. 功能清单（MVP 到 V1）

### MVP
- 反愿景/愿景MVP/身份声明录入
- 1年主线 / 1月 Boss / 每日任务结构化
- 今日任务完成与 XP 记录
- AI 纠偏提示（含目的论问题）

### V1
- 中断系统（定时/随机）
- 透镜视图（年/月/日）
- 反愿景常驻提示
- 连续链条与奖励

## 8. 成功指标（成功体验）

- 用户可以复述自己的身份声明与反愿景
- 每个任务能明确对齐主线目标
- 用户每天至少一次被“中断”并完成觉察
- XP 与连续链条形成持续行动

## 9. 对现有实现的定位

当前实现属于“概念展示阶段”。
- 已体现身份/AI/游戏化的文案方向
- 尚未实现：身份数据库、1‑Day Reset、层级任务、纠偏引擎、中断机制、透镜与力场结构

## 10. 平台扩展与形态规划

- 未来计划从 Web 扩展为 **iOS / Android App**（可能基于 pack 或其他开源方案转换）。
- 因此所有页面与组件必须保持 **跨平台可移植性与响应式**，避免对单一平台能力的硬绑定。

## 11. 未来开发准则（每次改动前检查）

- 这项功能是否让用户更接近“身份重塑”？
- 它是否形成可视化反馈回路？
- 是否让行动变成可见的“任务 + XP”？
- 是否提供了“反愿景/代价”的持续提醒？
