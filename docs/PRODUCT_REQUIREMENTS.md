# LifeOS 开发基本要求（长期约束）

> 本文档是所有开发的硬性约束。任何改动都必须对齐这些要求。

## 1. 国际化（必须）

- 系统必须支持三语：英语（en）、中文（zh）、法语（fr）。
- 文案统一使用 i18n 体系管理，不允许硬编码。\
- 采用可扩展的 copy 结构（按页面/模块拆分）。
- 语言选择必须**保存到本地缓存**并在启动时读取（保证跨会话一致体验）。

## 2. 多主题色系（必须）

- 必须支持**多套完整色板**，并可在 UI 中自由切换。\
- 任一主题应包含至少 10+ 个全局颜色 token（背景/表面/文本/边框/强调/危险/成功/警告/图表/状态等）。
- 切换主题后，**全站颜色同步替换**，不允许局部残留。
- 主题选择必须**保存到本地缓存**并在启动时读取（保证跨会话一致体验）。

## 3. 设计与布局（必须）

- 设计风格与布局必须持续参考 ui-ux-pro-max 的输出，并与产品愿景对齐。\
- 产品定位：身份优先 / 控制论反馈 / 游戏化驱动的 Life Console。
- 交互与信息层级必须强化「愿景 → 主线 → Boss → 今日任务」的结构隐喻。

## 4. 迭代原则（必须）

- 每次开发前先对照此文档与 `docs/PRODUCT_VISION.md`。
- 若新增功能不符合以上约束，必须先调整方案再实现。
## 4.1 数据库迁移约束（必须）

- **禁止**运行 `pnpm prisma migrate dev`。
- 只能使用 `pnpm prisma migrate deploy`（由你指定的流程执行）。
- 迁移采用**手写 SQL 脚本**并保存到 `prisma/migrations/`。

## 5. 固定内部路由（必须）

- `/specs`：开发基本要求（本文档）
- `/vision`：产品愿景（`docs/PRODUCT_VISION.md`）

## 6. 平台扩展（必须）

- 产品未来将扩展为 iOS / Android App（可能采用 pack 或其他开源方案转化）。
- 所有前端实现需保持跨平台可移植性：响应式布局、无浏览器专属 API 依赖、避免平台锁定式 UI。
